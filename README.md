# 原神记录站

一个纯静态网页项目，用于在本地记录和查看原神相关数据。项目不依赖构建工具，直接双击打开 `index.html` 即可使用。

## 页面组成

`index.html` 是整合入口，左侧导航提供两个页签：

| 页签 | 子页面 | 主要用途 |
| --- | --- | --- |
| 抽卡记录 | `index-wish.html` | 管理常驻池、限定角色池、限定武器池的抽卡数据，查看总览、池子统计、版本时间线和 5 星历史。 |
| 贵重资源 | `index-precious.html` | 管理祝圣之霜、启圣之尘等贵重资源的收入、支出、版本归属和剩余数量。 |

两个子页面都支持 JSON 导入导出，并会把当前数据暂存在浏览器 `localStorage` 中。刷新页面通常不会丢失当前数据，但长期备份仍应以“导出当前数据”生成的 JSON 文件为准。

## GitHub Pages 发布

仓库保持从 `main` 分支根目录发布，由 GitHub 的 `pages-build-deployment` 在推送后自动部署，无需切换为自定义 GitHub Actions。

iframe、CSS 和 JavaScript 使用固定的 `?v=` 版本号以利用浏览器和 CDN 缓存。修改相关页面或静态资源后，应同步更新对应 HTML 中的版本号，避免浏览器继续使用旧文件。

## 抽卡记录页

抽卡记录页用于维护三个池子的抽卡情况：

- 常驻池：统计常驻池总抽数、5 星记录、4 星角色和武器索引。
- 限定角色池：统计限定角色池出金、歪和 UP 情况，并支持捕获明光标记。
- 限定武器池：统计限定武器池出金、UP 情况和 4 星索引。

页面主体包含：

- 总览统计：展示总抽数、5 星数量、限定/常驻相关统计。
- 池子卡片：按池子展示总抽数、当前垫抽、平均出金间隔、4 星索引数量等。
- 版本时间线：展示限定池 5 星记录在版本维度上的分布。
- 历史列表：查看和编辑 5 星记录。

### 抽卡记录顶部操作栏

| 操作 | 作用 |
| --- | --- |
| 上传 JSON | 导入抽卡记录页原生 JSON。当前页面仅支持 `schemaVersion: 4`。 |
| 导入 UIGF | 在已加载主 JSON 的基础上导入 UIGF 文件，增量补充 4 星索引、5 星记录和总抽数。导入前会进行 offset 校验，校验失败时不能应用。 |
| 下载模板 | 下载空白抽卡记录模板 `wish-data.schema-v4.template.json`。 |
| 导出当前数据 | 将页面当前抽卡数据导出为 `wish-data.schema-v4.export.json`。页面编辑后应使用此按钮保存备份。 |
| 加载示例 | 加载内置示例数据，用于查看页面效果和数据结构。 |
| 清空本地数据 | 清除抽卡记录页保存在浏览器中的本地数据，不会删除已经导出的 JSON 文件。 |

顶部状态区还会显示：

- 当前数据：当前加载的数据来源或文件名。
- 当前数据已修改，尚未导出：提示页面数据已变化，需要手动导出。
- 同步状态：展示导入、导出、校验或错误信息。

## 贵重资源页

贵重资源页用于维护两类材料：

- 祝圣之霜
- 启圣之尘

页面支持按版本管理收入，也支持记录其他来源收入和圣遗物定制支出。支出记录会关联版本、材料、圣遗物套装、部位和主属性，页面会据此统计收入、支出和结余。

页面主体包含：

- 材料统计：展示每种材料的总收入、总支出和剩余数量。
- 版本管理：维护版本名称、排序键和分组。
- 收入记录：维护版本来源收入和其他来源收入。
- 支出记录：维护材料消耗、版本、套装、部位、主属性和备注。

### 贵重资源顶部操作栏

| 操作 | 作用 |
| --- | --- |
| 上传 JSON | 导入贵重资源页原生 JSON。当前页面仅支持 `schemaVersion: 1`。 |
| 下载模板 | 下载空白贵重资源模板 `precious-resources.schema-v1.template.json`。 |
| 导出当前数据 | 将页面当前贵重资源数据导出为 `precious-resources.schema-v1.export.json`。页面编辑后应使用此按钮保存备份。 |
| 加载示例 | 加载内置示例数据，用于查看页面效果和数据结构。 |
| 清空本地数据 | 清除贵重资源页保存在浏览器中的本地数据，不会删除已经导出的 JSON 文件。 |

顶部状态区同样会显示当前数据来源、未导出修改提示和操作状态。

## 本地使用

直接打开：

```text
index.html
```

也可以分别打开子页面：

```text
index-wish.html
index-precious.html
```

本项目是静态页面，不需要安装依赖或启动服务。

## 项目结构

```text
.
├── index.html                 # 整合入口
├── index-wish.html            # 抽卡记录页面
├── index-precious.html        # 贵重资源页面
├── assets/
│   ├── css/                   # 页面样式
│   └── js/
│       ├── wish/              # 抽卡页：状态、视图、编辑与导入导出
│       └── precious/          # 贵重资源页：数据、视图与编辑
└── docs/                      # 数据格式说明
```

脚本按 HTML 中的顺序加载，并共享页面级状态。新增功能时应优先放入对应职责文件，避免再次形成单个超大脚本；项目仍然不需要构建步骤。
